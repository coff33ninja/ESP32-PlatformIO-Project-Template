# Data Directory

This directory contains files that will be uploaded to the ESP32's SPIFFS (SPI Flash File System) or LittleFS filesystem.

## Purpose
- **Web assets**: HTML, CSS, JavaScript files for web interfaces
- **Configuration files**: JSON, XML, or text configuration files
- **Certificates**: SSL/TLS certificates for secure connections
- **Static content**: Images, fonts, or other static resources

## File Upload
Upload files to ESP32 filesystem using:
```bash
# PowerShell
.\build.ps1 uploadfs

# Make
make uploadfs

# PlatformIO CLI
pio run --target uploadfs
```

## Example Files
- `wifi_config.json` - WiFi configuration (generated by wifi tool)
- `web/` - Web interface files
- `certs/` - SSL certificates
- `config/` - Application configuration files

## File Size Limits
- **SPIFFS**: Default partition ~1.5MB (configurable in platformio.ini)
- **Individual files**: No specific limit, but consider available space
- **Total files**: Limited by partition size and filesystem overhead

## Best Practices
- **Compress files**: Use gzip compression for web assets
- **Optimize images**: Use appropriate formats and sizes
- **Minimize JSON**: Remove unnecessary whitespace
- **Test uploads**: Verify files upload correctly before deployment

## Accessing Files in Code
```cpp
#include <SPIFFS.h>

void setup() {
    // Initialize SPIFFS
    if (!SPIFFS.begin(true)) {
        Serial.println("SPIFFS Mount Failed");
        return;
    }
    
    // Read a file
    File file = SPIFFS.open("/config.json", "r");
    if (file) {
        String content = file.readString();
        file.close();
        Serial.println("Config: " + content);
    }
}
```
